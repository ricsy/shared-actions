# 贡献指南

欢迎提交 PR 添加新的通用工作流或 Action。

## 目录结构

```
.github/
├── actions/           # 可复用 Action
│   └── <name>/
│       └── action.yml
└── workflows/
    └── <category>/   # 按功能分类
        └── <workflow>.yml
```

## Workflow 规范

新增 Workflow 需满足以下标准：

### 头部注释

```yaml
# =============================================================================
# <工作流名称>
#
# 功能：<简短描述>
# 触发：<触发类型>
#
# 输入：
#   <input-name>: <描述>（默认 <默认值>）
#
# 前置要求：
#   1. <前提条件>
#
# 使用示例：
#   jobs:
#     <job-name>:
#       uses: owner/shared-actions/.github/workflows/<category>/<workflow>.yml@main
# =============================================================================
```

### 必填内容

| 项目        | 要求                                    |
|-----------|---------------------------------------|
| 文件头注释     | 包含功能、触发方式、输入参数、前置要求、使用示例              |
| inputs 定义 | 所有参数需设置 `required: false` 和 `default` |
| 步骤注释      | 步骤添加中文说明                              |

## Action 规范

新增 Action 需满足以下标准：

### 头部注释

```yaml
# =============================================================================
# <Action 名称>
#
# 功能：<简短描述>
#
# 输入：
#   <input-name>: <描述>（默认 <默认值>）
#
# 使用方式：
#   steps:
#     - uses: owner/shared-actions/.github/actions/<name>@main
#       with:
#         <input-name>: <值>
# =============================================================================
```

### 必填内容

| 项目         | 要求                                    |
|------------|---------------------------------------|
| 文件头注释      | 包含功能、输入参数、使用方式                        |
| inputs 定义  | 所有参数需设置 `required: false` 和 `default` |
| runs.steps | 必须包含可执行步骤                             |

## 通用规范

| 项目        | 说明                              |
|-----------|---------------------------------|
| 目录层级      | 推荐使用 `python/`、`publish/` 等已有分类 |
| 多 job 工作流 | 需在注释中说明各 job 职责                 |
